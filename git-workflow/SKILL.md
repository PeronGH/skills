# Git Workflow

- **Branching**:
  - Create a new branch (`<type>/<description>`) for substantial or risky changes.
  - Direct commits to `main`/`master` are acceptable for early-stage projects or low-risk edits.
- Commit frequently instead of batching large changes.
- Use conventional commit messages in the format `<type>(<scope>): <description>`.
